Overview

The BuildChangeOrder API generates professional change order PDFs via HTTP. It handles all layout, financial calculations, and document assembly — you provide the data, we return a base64-encoded PDF.

What's unique

  • Running contract total — Original Contract + Previous COs + This CO = New Contract Total, calculated and rendered automatically
  • Schedule impact tracking — Days added/removed and new completion date displayed as a warning banner
  • Dual signature blocks — Client approval and contractor authorization lines on every document
  • Reason classification — Client Request, Unforeseen Conditions, Design Revision, Error or Omission, Owner-Initiated, Other

Base URL

https://buildchangeorder.co/api/v1

Also available on RapidAPI

The BuildChangeOrder API is listed on the RapidAPI marketplace for easy subscription management and unified billing alongside your other APIs.

Authentication

All requests to protected endpoints require an API key passed in the X-API-Key header.

X-API-Key: bco_your_api_key_here
🔑 Get an API key instantly — no credit card required — via POST /register.

Key format

All BuildChangeOrder API keys are prefixed with bco_ followed by 32 alphanumeric characters. Keys are shown only once at registration — store them securely.

Errors

The API returns standard HTTP status codes. Error bodies are JSON with a message field.

CodeMeaning
200Success
201Created (registration)
400Bad request — missing or invalid fields
401Missing or invalid API key
429Rate limit or monthly quota exceeded

  "docs": "https://buildchangeorder.co/api/docs"

POST /register

POST /api/v1/register No authentication required

Creates a new API key linked to the provided email address. Each email can register once. Rate-limited to 5 registrations per IP per hour.

Request body

POST /generate

POST /api/v1/generate Requires X-API-Key

Generates a change order PDF and returns it as a base64-encoded string. Decrements your monthly quota by 1.

from.emailstringoptionalContractor email

to.namestringrequiredClient name
to.emailstringoptionalClient email
to.addressstringoptionalClient address
change_order.numberstringrequiredChange order number (e.g. CO-001)
change_order.datestringrequiredISO 8601 date (e.g. 2026-04-19)
change_order.projectstringoptionalProject name

original_contract.referencestringoptionalContract title or reference number

  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "X-API-Key":    "bco_your_key_here"
  },
  body: JSON.stringify({
    from: { name: "Acme Agency", email: "hello@acme.com" },
    to:   { name: "BigCo Corp",  email: "client@bigco.com" },

    impact: { days_added: 10, new_completion_date: "2026-05-25" },
    tax_rate: 8.5
  })
});

const data = await response.json();

// Decode and save PDF
const bytes  = atob(data.pdf);
const arr    = new Uint8Array(bytes.length);
for (let i = 0; i < bytes.length; i++) arr[i] = bytes.charCodeAt(i);
const blob   = new Blob([arr], { type: "application/pdf" });
const url    = URL.createObjectURL(blob);
const link   = document.createElement("a");
link.href    = url;
link.download = "CO-002.pdf";
link.click();

resp = requests.post(

        "change_order": {"number": "CO-002", "date": "2026-04-19"},

data = resp.json()
with open("CO-002.pdf", "wb") as f:
